# Концепция проекта: MP3 to Voice Telegram Bot

## Проблема

Telegram разграничивает аудиофайлы и голосовые сообщения. Когда пользователь отправляет `.mp3` файл — он отображается как обычный аудиофайл с плеером. Голосовые сообщения (Voice Messages) имеют особый UX: они воспроизводятся прямо в чате, поддерживают скорость воспроизведения ×1.5 / ×2, и более удобны для прослушивания коротких записей на мобильных устройствах.

Конвертация MP3 → OGG/OPUS вручную требует установки сторонних инструментов (ffmpeg, Audacity и т.д.), что недоступно большинству нетехнических пользователей.

## Решение

Telegram-бот, который принимает `.mp3` файл и возвращает его как голосовое сообщение (Voice Message) в формате OGG/OPUS. Пользователь отправляет файл — бот мгновенно конвертирует и отвечает голосовым сообщением. Никакой установки программ, никакой регистрации.

## Целевая аудитория

- Подкастеры и авторы аудиоконтента, которые хотят делиться фрагментами через Telegram
- Пользователи, которые хотят отправить записанный аудиофайл как голосовое сообщение
- Разработчики, тестирующие интеграцию Telegram Voice API
- Нетехнические пользователи, которым нужна простая конвертация без установки ПО

## Ключевые фичи MVP

1. **Приём MP3 файлов** — бот реагирует на Document с `mime_type: audio/mpeg`
2. **Конвертация MP3 → OGG/OPUS** — через pydub + ffmpeg (кодек libopus)
3. **Отправка голосового сообщения** — через `bot.send_voice()`, файл отображается как Voice Message в Telegram
4. **Очистка временных файлов** — скачанные и сконвертированные файлы удаляются сразу после отправки
5. **Базовая обработка ошибок** — понятные сообщения пользователю при неверном типе файла или ошибке конвертации
6. **Команда /start** — краткое приветствие с инструкцией

## Вне скоупа MVP

- База данных и история конвертаций
- Авторизация и пользовательские аккаунты
- Конвертация других форматов (WAV, FLAC, AAC и т.д.)
- Очередь задач и асинхронная обработка больших файлов
- Веб-интерфейс или API
- Статистика и аналитика
- Лимиты на размер файла (кроме стандартных ограничений Telegram — 20 МБ для ботов без Telegram Bot API Server)
- Уведомления и рассылки

## Технический стек и обоснование

| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| Язык | Python 3.11+ | Стандарт для Telegram-ботов, богатая экосистема |
| Telegram-фреймворк | aiogram 3.x | Современный async-фреймворк, активно поддерживается, нативный asyncio |
| Конвертация аудио | pydub + ffmpeg | pydub — удобный Python API, ffmpeg — де-факто стандарт конвертации аудио |
| Конфигурация | python-dotenv | Простое управление переменными окружения через .env |
| Деплой | Railway.com | PaaS с нативной поддержкой Python, автодетект через Nixpacks, простая настройка переменных окружения, бесплатный tier для MVP |
| Хранение | Нет (только tmp/) | MVP не требует персистентности данных |

**Почему aiogram 3.x, а не python-telegram-bot?**
aiogram 3.x построен на чистом asyncio, имеет современный API с роутерами и мидлварами, активно развивается. Для production-качества кода это более подходящий выбор.

**Почему pydub + ffmpeg, а не прямой вызов ffmpeg?**
pydub даёт удобный Python API и абстракцию над форматами. ffmpeg устанавливается как системная зависимость через `nixpacks.toml` — Railway поддерживает это нативно.

**Почему Railway, а не Docker/VPS?**
Railway избавляет от необходимости поддерживать инфраструктуру. Деплой — пуш в git. Переменные окружения настраиваются в UI. Nixpacks автоматически определяет Python-проект и устанавливает зависимости.

## Метрики успеха

MVP считается успешным, если:

1. Бот получает `.mp3` файл и отправляет его обратно как **Voice Message** (не аудиофайл) в течение **разумного времени** (до 30 секунд для файлов до 5 МБ)
2. После отправки ответа **временные файлы удалены** с диска
3. При отправке неверного типа файла бот отвечает **понятным сообщением** вместо ошибки
4. Бот **не падает** при ошибках конвертации — пользователь получает сообщение об ошибке
5. Проект **деплоится на Railway** через пуш в git без дополнительных шагов
