# План реализации: MP3 to Voice Telegram Bot

## Обзор

Проект реализуется итеративно, по одному этапу за раз. Каждый этап завершается рабочим состоянием проекта и git-коммитом. Переход к следующему этапу — только после подтверждения пользователя.

---

## Этап 1: Структура проекта и конфигурация

### Задачи
- [ ] Создать файловую структуру проекта согласно спецификации
- [ ] Создать `requirements.txt` с зависимостями (aiogram, pydub, python-dotenv)
- [ ] Создать `.env.example` с переменной `BOT_TOKEN`
- [ ] Создать `.gitignore` (обновить существующий: tmp/, .env, __pycache__ и т.д.)
- [ ] Создать пустые `__init__.py` файлы
- [ ] Создать директорию `tmp/` с `.gitkeep`

### Ожидаемый результат
Готовая файловая структура проекта. Все файлы созданы (пока пустые или с заглушками). Проект можно клонировать и понять структуру.

---

## Этап 2: Точка входа и базовый бот

### Задачи
- [ ] Реализовать `main.py` — точка входа приложения
  - Загрузка переменных окружения через python-dotenv
  - Инициализация Bot и Dispatcher из aiogram
  - Подключение роутера из `bot/handlers.py`
  - Запуск polling
- [ ] Реализовать `bot/__init__.py` — экспорты модуля
- [ ] Реализовать базовый `bot/handlers.py` — только хендлер `/start`
  - Роутер aiogram
  - Хендлер команды `/start` с приветствием

### Ожидаемый результат
Бот запускается локально (при наличии `BOT_TOKEN` в `.env`), отвечает на `/start` приветственным сообщением. Все остальные сообщения игнорируются.

---

## Этап 3: Конвертация аудио

### Задачи
- [ ] Реализовать `bot/converter.py` — логика конвертации
  - Функция `convert_mp3_to_ogg(input_path: str, output_path: str) -> None`
  - Использование pydub для загрузки MP3
  - Экспорт в OGG с кодеком libopus
  - Обработка исключений (файл не найден, ошибка ffmpeg и т.д.)

### Ожидаемый результат
Модуль конвертации готов и протестирован (вручную, юнит-тесты не в скоупе MVP). Функция принимает путь к MP3, создаёт OGG файл по указанному пути.

---

## Этап 4: Хендлер MP3 файлов

### Задачи
- [ ] Расширить `bot/handlers.py`
  - Хендлер для Document с `mime_type == "audio/mpeg"`
  - Скачивание файла во временную директорию `tmp/`
  - Вызов функции конвертации из `bot/converter.py`
  - Отправка результата через `bot.send_voice()`
  - Удаление временных файлов (MP3 и OGG) в блоке `finally`
  - Catch-all хендлер для остальных типов сообщений — ответ "Отправь мне .mp3 файл"

### Ожидаемый результат
Бот принимает MP3 файлы, конвертирует и отправляет обратно как Voice Message. Временные файлы удаляются. Неверные файлы — понятный ответ пользователю.

---

## Этап 5: Обработка ошибок

### Задачи
- [ ] Добавить обработку ошибок в хендлер MP3:
  - Ошибка скачивания файла — сообщение пользователю
  - Ошибка конвертации — сообщение пользователю
  - Ошибка отправки — логирование
- [ ] Добавить логирование через `logging` (стандартная библиотека Python)
  - Логировать старт бота
  - Логировать получение файла (имя, размер)
  - Логировать успешную конвертацию
  - Логировать ошибки
- [ ] Убедиться, что `tmp/` очищается даже при ошибках (блок `finally`)

### Ожидаемый результат
Бот не падает при ошибках, пользователь всегда получает понятный ответ. Логи помогают диагностировать проблемы.

---

## Этап 6: Railway деплой и финальная проверка

### Задачи
- [ ] Создать `nixpacks.toml` — указать ffmpeg как системную зависимость для Railway
  ```toml
  [phases.setup]
  nixPkgs = ["ffmpeg"]
  ```
- [ ] Создать `railway.toml` — явно задать команду запуска
  ```toml
  [deploy]
  startCommand = "python main.py"
  ```
- [ ] Создать `README.md`
  - Описание проекта
  - Требования для локального запуска (Python 3.11+, ffmpeg)
  - Инструкция по запуску локально
  - Инструкция по деплою на Railway (шаги: создать проект, подключить repo, задать `BOT_TOKEN`)
  - Описание переменных окружения
- [ ] Финальная проверка: пуш в ветку, убедиться что Railway собирает и запускает бота

### Ожидаемый результат
Проект полностью готов к деплою. Деплой = пуш в git. README даёт исчерпывающую инструкцию для нового разработчика.

---

## Итоговая структура проекта

```
sound-to-voice-bot/
├── bot/
│   ├── __init__.py
│   ├── handlers.py       # хендлеры aiogram
│   └── converter.py      # логика конвертации через pydub
├── docs/
│   ├── concept.md
│   └── plan.md
├── tmp/                  # временные файлы (в .gitignore)
│   └── .gitkeep
├── .env                  # не коммитится
├── .env.example
├── .gitignore
├── nixpacks.toml         # системные зависимости для Railway (ffmpeg)
├── railway.toml          # конфигурация деплоя Railway
├── main.py               # точка входа
├── requirements.txt
└── README.md
```

## Зависимости

Python (requirements.txt):
```
aiogram==3.x
pydub==0.x
python-dotenv==1.x
```

Системные зависимости (устанавливаются Railway через nixpacks.toml):
- `ffmpeg` — обязательно для pydub (кодек libopus)
